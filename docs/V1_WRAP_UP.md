# V1 完结状态说明

## 项目结论

当前项目的 V1 主线已经完成，已达到可交付、可部署、可实际游玩的状态。

这里的“完成”指的是：

- 核心玩法闭环已经打通
- 主要移动端交互已经收口
- 部署方案已经明确
- 线上实机验证已经完成

因此，项目当前阶段应定义为：

- `V1 完成态`
- `可继续维护`
- `可继续新迭代`

而不是继续视为“V1 还没做完的开发中项目”。

## 已完成范围

围绕 V1 目标，本项目已经完成以下主线能力：

- 账号体系：账号密码登录、一次性邀请码注册、会话持久化
- 挂机主线：挂机区域切换、离线收益领取、金币/经验/材料/装备掉落
- 养成系统：背包、装备穿戴、双饰品位、一键最优装备、战力重算
- 装备深化：强化、分解、材料合成、定向锻造、词缀重铸、装备机制说明卡
- 战斗与社交：主线 Boss、世界 Boss、区域解锁、排行榜、日志、好友祝福
- 属性与成长：基于总经验的非线性等级成长，理论上限 3000 级；`goldBonus`、`expBonus`、`dropBonus`、`crit`、`luck` 都已经真实接入结算或战斗
- 体验收口：中文优先界面、中英文切换、分页格子背包、标签页结构、滚动位置保持、移动端长页收缩
- 运营与部署：测试管理台、Postgres 数据库方案、轻量服务器部署手册、Hermes 一站式上线文档

从产品角度看，V1 需要的“登录 -> 挂机 -> 掉落 -> 装备 -> Boss -> 排行/日志 -> 部署上线”这一整条主链已经完整。

## 验证状态

当前版本已经完成以下层面的验证：

- 自动化验证：`test`、`lint`、`build` 已能通过
- 本地验证：本地开发、数据库迁移、种子初始化、主要页面加载都已跑通
- 数值与封测准备：模拟脚本、封测计划、经济 sweep 已补齐
- 部署验证：Postgres 路线、轻量服务器部署步骤、Hermes 一站式上线说明已整理完成
- 线上验证：线上实机验证已经完成

因此，V1 不是停留在“代码写完”，而是已经进入“可实际交付”的状态。

## 当前结论

从现在开始，项目状态建议统一定义为：

- V1 已完成
- 当前处于维护与迭代阶段
- 后续工作属于新迭代，而不是继续补 V1 主线

这意味着后面如果继续做：

- 新玩法
- 新内容量
- 新社交功能
- 新运营工具

都应视为 V1 之后的版本演进，而不是把当前版本定义成“还没做完”。

## 非阻塞尾项

当前仍然存在少量非阻塞尾项，但它们不构成 V1 未完成：

- 工作区里还有少量本地调试文件未清理，例如 `browser-test.mjs`、`check-db.js`、`test-login.js`
- 后续仍可根据真实玩家反馈继续微调数值、提示文案和交互细节
- 如果部署环境变化，部署文档可能还需要按真实服务器情况做小幅更新

这些事项属于维护项或运营项，不影响当前版本被定义为 `V1 完成态`。
